#1.Standard naming convention(mandatory)
# Filename : start or end with test_ or _test
# Function name should be started with test keyword
# Class name should start with Testxxxx

#2.pytest running
# pytest test_intro.py
# pytest .
# pytest test_intro.py -v (Verbose)
# pytest -k "increment"   (k for keyword)
# pytest test_intro.py::test_increment     (Note that test_idtro is file name, increment and test_increment is function name)

#3.markers
#  applying some metadata on test functions, classes
#  built-in markers:
#     - @pytest.mark.skip(skipping one particular testing function - (cuz maybe that function is not yet implemented by the developer and you only have boiler plate code, so you skip this.))
#     - @pytest.mark.skipif(if condn satisfied skip it - maybe you want to skip a feature which doesnt work on windows if we have windows os(say))
#     - @pytest.mark.parameterize
